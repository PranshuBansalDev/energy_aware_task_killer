Weekly Updates

Week 5 Update (2/6 - 2/10):

Achieved This Week:

  • Met with Salma and Prof. Srivastava and converged on final project idea.
  • Acquired hardware (Nexus 4 smartphone).
  • Began initial project planning.
  • Setup Nexus 4 emulator on Android Studio.
  • Studied Android's HAL (Hardware Abstraction Layer) documentation (https://source.android.com/devices/) and (https://source.android.com/devices/halref/annotated.html) to see how to access data from relevant peripherals.
  • Searched for methods to measure energy consumed by Android. (http://dl.acm.org/citation.cfm?doid=2483760.2483780) and (https://source.android.com/devices/tech/power/index.html).
  • Examined documentation related to building Android Open Source Project (AOSP) in case we need to modify the kernel.


To Do In Week 6:

  • Select best applications to use for testing and data collection purposes.
  • Understand what phone metrics we will need to gather to implement the app.
  • Begin design of UI.
  • Learn more about AOSP.

Week 6 Update (2/13 - 2/17):

Achieved This Week:

  • Flashed the Nexus 4 device with stock 4.1.1 Android OS.
  • Began downloading AOSP repository.
  • Began reading documentation regarding AOSP.
  • Selected applications (Spotify, Google Maps, Angry Birds) as test applications.
  • Began development using Tasker API and Android Studio.


To Do In Week 7:

  • Attempt to Root phone to give full access to kill apps.
  • Leverage Tasker API to setup profiles to kill apps.
  • Finish up design of UI and have ready for final application.

Week 7 Update (2/20 - 2/24):

Achieved This Week:

  • Completed first design of UI.
  • Compiled kernel.
  • Flashed AOSP Mako image (without modifications) onto Nexus 4 Device.
  • Gained root access on Mako image.
  • Found out how to use Tasker to control killing and scheduling apps.


To Do In Week 8:

  • Find out how to give our app system app status to use pm disable/enable commands in Android Studio (so app is killed & NOT restarted).
  • Modify UI to add seekbar instead of checkboxes.

Week 8 Update (2/27 - 3/3):

Achieved This Week:

  • Extended our application functionality to include all user-installed launchable apps.
  • Installed our app as a system app.
  • Realized that AOSP's Mako build for our phone crashed once we attempted to make our application a system app.
  • Modified UI to have seekbars in a table next to each application.
  • Understood that apps were restarting after sending a kill command due to the "activity life-cycle" (https://developer.android.com/guide/components/activities/activity-lifecycle.html)


To Do In Week 9:

  • Re-flash phone with stock image from Android Developers (Android 5.1.1, API 21).
  • Re-root phone with new stock v5.1.1 Android Image.
  • Move towards custom app to address various issues with the Tasker framework.
  • Learn about methods to kill apps in a way that prevents them from restarting.

Week 9 Update (3/6 - 3/10):

Achieved This Week:

  • Re-flashed phone with new stock image.
  • Rooted phone using better version of TWRP recovery
  • Backed up phone contents.
  • Performed experimentation to prove that apps can be terminated via command line interactions (shell script).
  • Finished adding data structures to support killing applications at 3 different priority-driven battery levels.


To Do In Week 10:

  • Add dynamic library to seekbar that allows seekbar value to be displayed and updated real time.
  • Add Broadcast Receiver to monitor battery conditions and execute our shell script (figure out how to set this script up on phone).
  • Create new Activity in code that handles processing in separate Intent (to clean up code, and add buttons to UI).

Week 10 Update (3/13 - 3/17):

Achieved This Week:

  • Added dynamic seekbar library and integrated so seekbar values are displayed real-time in UI.
  • Added STOP and RUN buttons for execution so that user can control ON/OFF state of application.
  • Created Processing Activity in Android Studio and wrote logic to display applications being killed to screen.
  • Finished Broadcast Receiver logic, and started testing it in various edge cases to ensure it works.
  • Installed "BusyBox" in order to access shell commands such as "awk".


To Do In Finals Week:

  • Finish de-bugging boundary conditions related to Broadcast Receiver.
  • Figure out way to bypass permission issues related to storing shell script in sdcard of Android.
  • Improvements to UI.

Finals Week Update (3/20 - 3/24):

Achieved This Week:

  • Realized that broadcast receiver class gets recycled after each call to "onReceive()" returns.
  • Made adjustments to broadcast receiver to allow persistence of previous state of input parameters.
  • App permissions issues resolved.
  • App is completely functional.
  • Finishing touches (some changes to UI).
